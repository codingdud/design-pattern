# Dont Repeate youself:
aimed at reducing redundancy and improving code maintainability.
Every piece of knowledge should have a single, authoritative representation in your system. Don't duplicate logic, data, or intent.
# why DRY matters?
- code resuability: make resuable function for repetative task
- maintainlibilty: make chages at single place 
- error reduction : make cahge in single place
